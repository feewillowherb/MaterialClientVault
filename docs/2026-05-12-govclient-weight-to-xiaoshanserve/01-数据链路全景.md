# 01 - 数据链路全景

## 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│  阶段 1: 硬件采集层                                             │
│  DiBangScaleReceiver (BLL/DiBang/DiBangScaleReceiver.cs)          │
│  SerialPort → 多协议解析(耀华/托利多/卓源) → ScaleData 事件      │
└─────────────────────────────────────────────────────────────────┘
                              ↓ DataReceived 事件
┌─────────────────────────────────────────────────────────────────┐
│  阶段 2: 稳定性检测层 (Form1.weightTimerEvent)                   │
│  Timer(2000ms) → 比较 nowWeight vs Weight → 计数器递减           │
│  weightMaxCount == 0 时视为稳定                                  │
└─────────────────────────────────────────────────────────────────┘
                              ↓ UpLoadWeightDate()
┌─────────────────────────────────────────────────────────────────┐
│  阶段 3: 本地入库层                                             │
│  Form1.UpLoadWeightDate() → mRePushWeight → bRePushWeight.Add()  │
│  SQLite (RePushWeight 表)                                        │
└─────────────────────────────────────────────────────────────────┘
                              ↓ Timer(5000ms) 轮询
┌─────────────────────────────────────────────────────────────────┐
│  阶段 4: 上报到 XiaoShanServe                                    │
│  ServicePushWeightToGovApi.agentTmer_Main_Elapsed()              │
│  SQLite 查询未同步记录 → 图片 Base64 编码 → HTTP POST             │
│  POST {govXiaoShanUrl}/post                                      │
└─────────────────────────────────────────────────────────────────┘
                              ↓ /Api/Post
┌─────────────────────────────────────────────────────────────────┐
│  阶段 5: XiaoShanServe 接收与转发                                │
│  ApiController.Post() → 验证 buildLicenseNo → 存入 SQLite       │
│  ExplortStatisticBgService (5秒定时) → 转发到政府平台            │
│  POST http://191.12.15.58:8899/sapi/v1/inoutRecord/save        │
└─────────────────────────────────────────────────────────────────┘
```

## 核心组件清单

### GovClient（客户端）

| 组件 | 文件 | 职责 |
|------|------|------|
| `DiBangScaleReceiver` | `BLL/DiBang/DiBangScaleReceiver.cs` | 多协议串口接收器（耀华/托利多/卓源） |
| `WeightDevice` | `BLL/WeightDevice.cs` | 旧版串口设备（HEX/String 两种模式） |
| `Form1` | `Form1.cs` | **主窗体** — 重量稳定性检测、车牌识别回调、数据上报、LED 控制 |
| `CaptureDevice` | `BLL/CaptureDevice.cs` | 海康/臻识摄像头抓拍与车牌识别 |
| `ServicePushWeightToGovApi` | `Services/ServicePushWeightToGovApi.cs` | 定时轮询 SQLite 上报到政府平台 |
| `ServiceTelnet` | `Services/ServiceTelnet.cs` | Telnet 网络连通性检测 |
| `bRePushWeight` | `Lib/CodeSmith/BLL/RePushWeight.cs` | SQLite 数据访问层 |
| `mRePushWeight` | `Model/CodeSmith/RePushWeight.cs` | 待上报数据实体 |

### XiaoShanServe（服务端）

| 组件 | 文件 | 职责 |
|------|------|------|
| `ApiController` | `Controllers/ApiController.cs` | 数据接收端点 `/Api/Post` |
| `ExplortStatisticBgService` | `Service/ExplortStatisticBgService.cs` | 后台定时转发到政府平台 |
| `GovSyncData` | `Models/GovSyncData.cs` | 同步数据实体（SqlSugar） |
| `GovProject` | `Models/GovProject.cs` | 项目实体（接入码管理） |
| `DbHelper` | `Common/Core/DbHelper.cs` | SQLite 数据库操作 |

## 事件与定时器

| 触发器 | 间隔 | 动作 |
|--------|------|------|
| `DiBangScaleReceiver.DataReceived` | 实时（串口数据到达） | 更新 `_weight` 字段、刷新 UI 显示 |
| `weightTimer` | 2000ms | 稳定性检测 + 车牌识别结果处理 + 上报 |
| `timer`（心跳） | 3000ms | 设备在线状态检测、地磅重连 |
| `agentTmer_Heart` | 59000ms | 联通模式下的心跳包 |
| `agentTmer_Main` | 5000ms | 轮询 SQLite 未同步记录并上报 |
| `ExplortStatisticBgService` | 5000ms | XiaoShanServe 转发数据到政府平台 |

## 数据实体关系

```
GovClient 端:
mRePushWeight (本地 SQLite)
├── CarNo           车牌号
├── CarNoColor      车牌颜色
├── GrossWeight     毛重 (decimal)
├── TareWeight      皮重
├── SnapTime        称重时间
├── SnapImages      本地图片路径(逗号分隔)
├── BuildLicenseNo  场地标识
├── EquipmentNumber 设备编号
├── SyncStatus      同步状态 (0=未同步, 1=失败, 2=成功)
├── FailCount       失败次数
├── FailReason      失败原因枚举
└── FailMsg         失败信息

         ↓ HTTP POST (图片转 Base64)

XiaoShanServe 端:
mGovRequestWeight (API 请求模型)
├── carNo           车牌号
├── carNoColor      车牌颜色
├── buildLicenseNo  场地标识
├── goodsWeight     货物重量
├── snapTime        称重时间
├── snapImages[]    Base64 图片数组
├── carType         车型(大车/小车)
├── deviceID        设备编号(01进/02出)
└── siteType        场地类型(1=工地)

         ↓ 存储

GovSyncData (SQLite, SqlSugar)
├── CarNo, goodsWeight, snapTime
├── BuildLicenseNo, sourceData(JSON)
├── snapImages(本地路径)
├── SyncType(0=待同步, 1=成功, 2=失败)
└── SyncNumber(同步次数, 最大5次)
```
