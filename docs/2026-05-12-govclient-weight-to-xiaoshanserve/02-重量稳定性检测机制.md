# 02 - 重量稳定性检测机制

## 概述

`Fdsoft.Weight.GovClient` 中**没有独立的 `WeightStabilityHelper` 类**。重量稳定性检测逻辑**完全内联在 `Form1.weightTimerEvent()` 方法中**，通过一个简单的计数器机制实现。

## 算法实现

**文件**: `Form1.cs`，方法 `weightTimerEvent()`

### 核心变量

```csharp
// Form1 字段
decimal _weight = 0;        // 当前重量（毛重 - 皮重）
decimal nowWeight = 0;     // 本轮最大重量
int weightMaxCount = 5;     // 稳定计数器（倒计数）
object weightLock = new object();  // 同步锁
```

### 检测流程

```
weightTimer (每 2 秒触发)
    │
    ├── 检查是否有车牌识别结果 (captureResult != null)
    │
    ├── 读取 minWeight 配置阈值
    │
    ├── if (Weight > minWeight):
    │       ├── if (nowWeight < Weight):
    │       │       nowWeight = Weight     // 更新最大重量
    │       │       weightMaxCount = 5     // 重置计数器（未稳定）
    │       └── else:
    │               weightMaxCount -= 1   // 重量不再增大，计数器递减
    │
    └── else:
            nowWeight = 0          // 重量不足，清零
            weightMaxCount = 5      // 重置计数器

    if (weightMaxCount == 0):
        // 稳定！触发上报
        UpLoadWeightDate(captureLicense, licenseColor, nowWeight, imgPathList)
        nowWeight = 0
        weightMaxCount = 5
```

### 伪代码

```
稳定条件 = 连续 5 次 Timer tick（共 10 秒）重量不再超过当前最大值
```

## 配置参数

| 参数 | 来源 | 默认值 | 说明 |
|------|------|--------|------|
| `minWeight` | `App.config["minWeight"]` | 配置文件 | 最小称重阈值（KG），低于此值不触发检测 |
| Timer 间隔 | 硬编码 | 2000ms | `weightTimer.Interval` |
| 稳定次数 | 硬编码 | 5 次 | `weightMaxCount` 初始值 |
| 实际稳定时间 | 计算 | ~10 秒 | 5 次 × 2 秒 = 10 秒 |

## 重量数据来源

### 串口读取链路

```
SerialPort.DataReceived (实时)
    │
    ▼
DiBangScaleReceiver.SerialPort_DataReceived()
    │
    ├── 协议自动识别/强制指定
    ├── 解析重量值 + 小数位数
    │
    ▼
DataReceived 事件 → Form1 事件处理器
    │
    ├── decimal.Parse(weightStr) → weightValue
    ├── 单位转换 (KG→吨: ×1000)
    ├── 更新 UI: GrossWeight.Text
    ├── 计算: _weight = weightValue - tareValue
    └── nowWeight/weightMaxCount 在下一次 Timer 中使用
```

### 支持的协议

`DiBangScaleReceiver` 支持 4 种协议：

| 协议 | 枚举值 | 停止位 |
|------|--------|--------|
| 耀华 XK3190-A9 TF0 | `DiBangProtocolType.耀华XK3190A9TF0` | 1 |
| 嘉兴丹柯 DS8 托利多模式1 | `DiBangProtocolType.嘉兴丹柯DS8托利多模式1` | 1 |
| 嘉兴丹柯 DS8 卓源模式 | `DiBangProtocolType.嘉兴丹柯DS8卓源模式` | 2 |
| 未知（自动识别） | `DiBangProtocolType.未知协议` | 1 |

## 与 MaterialClient 的对比

| 维度 | GovClient | MaterialClient |
|------|-----------|---------------|
| **稳定性算法** | 计数器：连续 5×2s 不超最大值 | 时间窗口分析：Buffer + 范围阈值 |
| **判定精度** | 仅判断"不再增大" | 判断范围 (Max-Min) ≤ 阈值 |
| **稳定值** | 取最大值 (nowWeight) | 取 (min + max) / 2 |
| **响应速度** | ~10 秒（5×2s） | ~3 秒（窗口 3000ms） |
| **实现方式** | Timer + 字段 | Rx.NET Buffer + DistinctUntilChanged |
| **并发安全** | Monitor 锁 | Rx 调度器 |
| **可配置性** | 仅 minWeight 可配 | 4 个参数均可配 |
| **代码组织** | 内联在 Form1 | 独立的 CreateStabilityStream() |

## 潜在问题

1. **仅判断"不超最大值"**：无法检测重量在稳定值附近波动的情况（如 5000→5010→4990→5005），只要不超过当前最大值就计数递减。
2. **重量下降不会重置**：如果重量从 5000 降到 4990（仍 > maxWeight），计数器仍递减，可能误判为稳定。
3. **Timer 精度问题**：`System.Timers.Timer` 在 WinForms 中可能受 UI 线程阻塞影响，实际间隔不稳定。
4. **无数据点数量要求**：不像 MaterialClient 要求最少 8 个有效数据点，这里只要有 5 个 tick 就判定稳定。
5. **无状态机**：没有 OffScale/WaitingForStability 等状态概念，只有简单的"稳定/不稳定"二元判断。
