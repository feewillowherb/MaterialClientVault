# 01 - 源项目称重与同步组件分析

> 源项目：`repos/Fdsoft.Weight.GovClient/Fdsoft.Weight.GovClient/`

## 1. 称重数据采集

### 1.1 地磅串口通信

**文件**: `BLL/WeightDevice.cs`

通过 `System.IO.Ports.SerialPort` 与地磅设备通信，支持两种接收模式：

| 模式 | 枚举值 | 说明 |
|------|--------|------|
| HEX | `ReceType.HEX` | 固定 12 字节，检查起始符 `0x02` 和结束符 `0x03` |
| String | `ReceType.String` | 按结束符 `=` 读取，接收后反转字符串 |

**支持的协议**（`App.config` → `equipmentTFType`）：

| 协议 | 模式 | 字节数 |
|------|------|--------|
| 耀华TF0 | HEX | 12 |
| 嘉兴丹柯DS8耀华模式2 | HEX | 12 |
| 其他（默认） | String | 结尾 `=` |

**数据流**：`SerialPort.DataReceived` → `ReceHEX/ReceString` → `SendBuffer/SendString` 事件

**配置项**（`App.config`）：
- `equipmentCOM`：串口号（如 `COM1`）
- `equipmentRate`：波特率（如 `9600`）
- `equipmentTFType`：协议类型

### 1.2 大磅数据接收器

**文件**: `BLL/DiBangScaleReceiver.cs`

更完善的多协议串口接收器，支持耀华/托利多/卓源等协议。在 `Form1.cs` 中使用，通过 `DataReceived` 事件将重量数据传递给主窗体。

### 1.3 重量稳定性检测

**位置**: `Form1.weightTimerEvent()`（内联在主窗体中）

- Timer 间隔：2000ms
- 判定逻辑：连续 5 次 Timer tick 重量不再增大（`nowWeight <= Weight`），则视为稳定
- 稳定后触发 `UpLoadWeightDate()` 将数据写入 SQLite

## 2. 图像采集与处理

### 2.1 车牌识别设备

**文件**: `BLL/CaptureDevice.cs`

支持两种设备：

| 设备 | 枚举值 | SDK |
|------|--------|-----|
| 海康威视 | `deviceType.海康威视` | CHCNetSDK（`NET_DVR_StartListen_V30`） |
| 臻识科技 | `deviceType.臻识科技` | VzLPRSDK（`VzLPRClient_Open`） |

**海康威视回调处理**：
- `COMM_UPLOAD_PLATE_RESULT`（老版本）→ `ProcessCommAlarm_Plate`
- `COMM_ITS_PLATE_RESULT`（新版本）→ `ProcessCommAlarm_ITSPlate`

**臻识科技回调处理**：
- `OnCommonRecv` → JSON 解析 → 提取车牌号和颜色

**图片保存**：
- 保存路径：`/picture/{yyyy-MM-dd}/ITS_Pictype_PicNum{n}.jpg`
- 压缩阈值：`CompressImage` 配置项（默认 200KB）
- 压缩方法：`CaptureDevice.CompressImage()` — 缩放 50% + JPEG 质量递减

**配置项**：
- `deviceType`：设备类型（0=海康，1=臻识）
- `captureIP` / `capturePort`：监听地址
- `VideoIP` / `VideoPort` / `VideoUserName` / `VideoPwd`：设备连接参数

### 2.2 图片在同步中的使用

### 3.4 同步执行流

```
Timer(5000ms) 触发
  → 检查 GovApiOnline（由 ServiceTelnet 维护）
  → 查询 SQLite: SyncStatus <> 不再上传 AND SyncStatus <> 成功
  → 逐条处理:
    → 检查 RSA 授权时间
    → GetImages(): 读取本地图片 → Base64 编码
    → 构建 mGovRequestWeight（根据网络公司选择字段）
    → HTTP POST 到 {govXiaoShanUrl}/post
    → 解析 mGovResponseBase：code=200 为成功
    → 更新 SyncStatus/FailCount/FailReason
    → SaveToSqlLite(): FailCount >= 9 && 非网络原因 → 标记"不再上传"
```
