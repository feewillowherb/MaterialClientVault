## 3. 替代方案

### 3.1 JWT Token 方案

如果不使用 PFX 方案，可以考虑 JWT Token：

```csharp
// JWT Token 验证
public class JwtTokenValidator
{
    public bool ValidateToken(string token, out ClaimsPrincipal? principal)
    {
        try
        {
            var handler = new JwtSecurityTokenHandler();
            var validationParameters = new TokenValidationParameters
            {
                ValidateIssuerSigningKey = true,
                IssuerSigningKey = new SymmetricSecurityKey(Encoding.UTF8.GetBytes(_secret)),
                ValidateIssuer = true,
                ValidIssuer = "UrbanManagement",
                ValidateAudience = true,
                ValidAudience = "UrbanClient",
                ValidateLifetime = true
            };

            principal = handler.ValidateToken(token, validationParameters, out _);
            return true;
        }
        catch
        {
            principal = null;
            return false;
        }
    }
}
```

**优势**：
- 实现简单，无需证书管理
- 性能优于证书验证
- 支持无状态认证

**劣势**：
- 安全性低于 PFX 方案
- 缺乏双向认证机制
- 密钥管理复杂性

### 3.2 API Key 方案

```csharp
// API Key 验证
public class ApiKeyValidator
{
    public async Task<bool> ValidateApiKeyAsync(string apiKey)
    {
        var hashedKey = HashApiKey(apiKey);
        var exists = await _dbContext.ApiKeys.AnyAsync(k => k.KeyHash == hashedKey);
        return exists;
    }

    private string HashApiKey(string apiKey)
    {
        using var sha256 = SHA256.Create();
        var bytes = Encoding.UTF8.GetBytes(apiKey);
        var hash = sha256.ComputeHash(bytes);
        return Convert.ToBase64String(hash);
    }
}
```

**优势**：
- 实现简单，易于理解
- 运维成本低
- 适合内部服务

**劣势**：
- 安全性有限
- 缺乏细粒度权限控制
- 密钥轮换复杂

### 3.3 OAuth 2.0 方案

```csharp
// OAuth 2.0 授权
public class OAuthAuthorizationService
{
    public async Task<string> GetAuthorizationTokenAsync()
    {
        var client = new HttpClient();
        var request = new HttpRequestMessage(HttpMethod.Post, _tokenEndpoint);
        request.Content = new FormUrlEncodedContent(new[]
        {
            new KeyValuePair<string, string>("grant_type", "client_credentials"),
            new KeyValuePair<string, string>("client_id", _clientId),
            new KeyValuePair<string, string>("client_secret", _clientSecret),
            new KeyValuePair<string, string>("scope", "urbanapi")
        });

        var response = await client.SendAsync(request);
        var tokenResponse = await response.Content.ReadFromJsonAsync<TokenResponse>();
        return tokenResponse?.AccessToken ?? string.Empty;
    }
}
```

**优势**：
- 标准化协议，生态成熟
- 支持多种授权模式
- 适合第三方集成

**劣势**：
- 架构复杂度高
- 需要授权服务器
- 实现成本较高

### 4.2 最终推荐

**推荐方案：PFX 证书 + 许可证号双重验证**

**理由**：
1. **安全性最高**：提供双向认证、传输加密、数字签名
2. **兼容性好**：可以保留现有的 BuildLicenseNo、FdBuildLicenseNo 验证
3. **可扩展性强**：支持未来功能扩展和升级
4. **标准化程度高**：基于成熟的 SSL/TLS 协议

**实施策略**：
- 采用渐进式实施，降低风险
- 保留向后兼容性，确保平滑过渡
- 建立完善的监控和运维机制
- 提供详细的文档和培训
