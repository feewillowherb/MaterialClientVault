## 3. 核心端点分析

### 3.1 数据接收端点 `POST /Api/Post`

**源码位置**: `Controllers/ApiController.cs`

**请求模型** (`mGovRequestWeight`):

| 字段 | 类型 | 说明 | 必填 |
|------|------|------|------|
| carNo | string | 车牌号 | 是 |
| carColor | string | 车身颜色 | 否 |
| carNoColor | string | 车牌颜色（黄/蓝） | 是 |
| buildLicenseNo | string | 场地标识（城管对接码） | 条件 |
| fdBuildLicenseNo | string | 凡东接入码（MD5） | 条件 |
| inOutType | int | 进出类型（0=进/1=出） | 是 |
| equipmentNumber | string | 地磅编号 | 是 |
| equipmentType | string | 地磅型号 | 是 |
| grossWeight | int | 毛重 | 是 |
| tareWeight | int | 皮重（无则为0） | 是 |
| snapTime | string | 称重时间 | 是 |
| snapImages | string[] | Base64 图片数组 | 是 |
| carType | string | 车型（大车/小车） | 否 |
| deviceID | string | 设备编号（01进/02出） | 否 |
| siteType | string | 场地类型（1=工地/2=消纳场） | 否 |
| goodsWeight | string | 货物重量（千克） | 否 |

**业务流程**:

```
1. 验证接入码
   ├── fdBuildLicenseNo 不为空 → 查 Gov_Project 匹配 → 赋值 buildLicenseNo + ProId
   └── buildLicenseNo 不为空 → 查 Gov_Project 匹配 → 赋值 ProId
   └── 两者都为空 → 拒收

2. 处理图片
   ├── 遍历 snapImages 数组（Base64 字符串）
   ├── 保存到 FilesPhysicalPath/TempUpload/{buildLicenseNo}/{ticks}_{i}.jpg
   └── 超过 CompressImage KB 自动压缩（ImageQuality=60L）

3. 构建 GovSyncData 实体
   ├── 如果 grossWeight > 0，覆盖 goodsWeight
   ├── sourceData = 原始 JSON（snapImages 已清空）
   ├── snapImages = 逗号分隔的本地图片路径
   ├── SyncType = 0（待同步）
   └── SyncNumber = 0

4. 插入 SQLite → 返回成功/失败
```

**响应格式** (`ApiResultDto`):

```json
{
  "success": true/false,
  "msg": "成功/失败原因",
  "code": 200/-1,
  "data": null
}
```

## 4. 后台同步服务

**源码位置**: `Service/ExplortStatisticBgService.cs`

### 执行逻辑

```
BackgroundService.ExecuteAsync → while (!cancelled)
    ├── DoExecuteAsync()
    │   ├── 查询同步开启且已配置对接码的项目
    │   ├── 查询待同步数据：SyncType != 1 且 SyncNumber < 5
    │   └── foreach (record):
    │       ├── GetImages() 读取本地图片 → Base64
    │       │   └── 图片不存在 → 标记 SyncNumber=10（不再同步）
    │       ├── 构建 mGovRequestWeight
    │       │   └── carType 判定：goodsWeight > 4500 → "大车"
    │       ├── HTTP POST 到 GovAddress
    │       ├── 成功 → SyncType = 1, 更新 LastSyncTime
    │       ├── 失败 → SyncType = 2, SyncNumber += 1
    │       └── 异常 → SyncNumber = 10（不再同步）
    └── await Task.Delay(5000)
```

### 关键配置项

| 配置键 | 默认值 | 说明 |
|--------|--------|------|
| GovAddress | `http://191.12.15.58:8899/sapi/v1/inoutRecord/save` | 政府 API 地址 |
| FilesPhysicalPath | `D:\Findong.XiaoShan\` | 图片存储根路径 |
| CompressImage | `200` | 图片压缩阈值（KB） |

## 6. 公共组件

### FdController 基类

所有 Controller 的基类，提供：
- `result` 字段：`ApiResultDto` 默认实例
- `JsonDate()` / `JsonDateTime()`：统一 JSON 序列化（自定义日期格式）
- `CompressImage()`：图片压缩方法

### DbHelper

静态数据库访问工具：
- `GetDb()` → 返回 `SqlSugarClient` 实例
- 通过 `DbExtension.AddDb()` 在 Startup 中注册

### AppSettings

静态配置读取封装：
- `GetAppConfig<T>(key)` → 从 `IConfiguration` 读取配置值
- `Bind<T>()` → 绑定配置到对象

### ApiResultDto

统一响应模型：

```csharp
public class ApiResultDto
{
    public bool success { get; set; }
    public object data { get; set; }
    public string msg { get; set; }
    public int code { get; set; } = 200;
    public int count { get; set; }
}
```

## 7. 与迁移相关的架构特点

### 需要保留的行为
1. **接入码双重验证**：支持 `buildLicenseNo` 和 `fdBuildLicenseNo` 两种验证方式
2. **图片 Base64 → 文件存储**：接收 Base64 图片，保存到本地，记录路径
3. **5 秒轮询同步**：待同步数据定时转发到政府 API
4. **重试机制**：最多 5 次重试，超过后标记为不再同步
5. **同步日志记录**：每次同步操作记录详细日志

### 可以优化的行为
1. **图片存储**：从逗号分隔路径改为 `AttachmentFile` 关联体系
2. **ORM**：SqlSugar → EF Core，获得强类型 LINQ 支持
3. **HTTP 客户端**：`HttpWebRequest` → `HttpClient`/Refit + Polly
4. **后台服务**：自定义 `BackgroundService` → ABP `AsyncPeriodicBackgroundWorkerBase`
5. **错误处理**：全局 try-catch → ABP 异常处理中间件
6. **配置管理**：静态 `AppSettings` → ABP `IOptions<T>` + Settings Provider
