# 03 - 数据模型映射与迁移

> **更新说明**：本文档已按最新决策更新：核心实体主键统一为 `Guid`，并采用“重建新库 + 旧数据迁移”方案。

## 1. 实体映射总览

### 1.1 实际实现策略

当前采用“**新库重建 + 数据迁移**”策略：
- 核心实体主键统一为 `Guid`
- 放弃原库直接附加，改为导入旧数据到新库
- 旧字段在新库中尽量保留语义，必要时通过映射转换

### 1.2 Gov_Project → GovProject

| 旧字段（SqlSugar） | EF Core 属性 | 类型 | 变更 | 说明 |
|-------------------|-------------|------|------|------|
| *(无)* | **Id** (Guid) | 新增 | PK 改为 Guid | ProId 即为 Id，不再单独字段 |
| ProId | *(合并到 Id)* | 删除 | ProId = Id | 简化设计 |
| ProName | ProName | string | 不变 | |
| BuildLicenseNo | BuildLicenseNo | string? | 不变 | |
| FdBuildLicenseNo | FdBuildLicenseNo | string? | 不变 | |
| SyncStatus | SyncStatus | bool? | 不变 | 可空类型 |
| LastSyncTime | LastSyncTime | DateTime? | 不变 | |
| DeleteStatus | DeleteStatus | bool? | 不变 | **未重命名为 IsDeleted** |
| AddTime | AddTime | DateTime? | 不变 | **未改为 CreationTime** |

**与原规划的关键差异**：
1. **主键类型**：原规划 `int`，实际 `Guid` — ProId 即主键，简化数据模型
2. **字段名保留**：原规划建议重命名（`DeleteStatus→IsDeleted`、`AddTime→CreationTime`），实际全部保留旧名
3. **无审计接口**：原规划建议 `IHasCreationTime`，实际未采用

> 说明：`GovSyncData` 是由 `UrbanWeighingRecord` 上行到远程服务时构建的数据模型，仅承载业务数据载荷，不承载同步状态字段。

**与原规划的关键差异**：
1. `UrbanWeighingRecord` 是服务端接收客户端上报的原始称重数据
2. `GovSyncData` 是服务端向远程平台上报时的同步数据
3. 二者字段保持高度一致，但 `GovSyncData` 不持有同步状态字段
4. `SyncTime` 语义明确为“上传时的时间”

- 这是**全新实体**，不在旧系统中存在

- 图片采用 `AttachmentFile` 关联方案（不使用 OSS）

## 2. 目标 EF Core 配置

```csharp
// UrbanManagementDbContext.cs OnModelCreating（实际代码）

// Gov_Project 表
builder.Entity<GovProject>(b =>
{
    b.ToTable("Gov_Project");
    b.HasKey(e => e.Id);                         // Guid 主键
    b.Property(e => e.ProName).IsRequired().HasMaxLength(200);
    b.Property(e => e.BuildLicenseNo).HasMaxLength(200);
    b.Property(e => e.FdBuildLicenseNo).HasMaxLength(200);
    // 无额外索引配置（原规划建议对 BuildLicenseNo/FdBuildLicenseNo 建索引）
});

// Gov_SyncData 表
builder.Entity<GovSyncData>(b =>
{
    b.ToTable("Gov_SyncData");
    b.HasKey(e => e.Id);                         // Guid 主键
    b.Property(e => e.CarNo).HasMaxLength(50);
    b.Property(e => e.GoodsWeight).HasMaxLength(50);
    b.Property(e => e.SnapTime).HasMaxLength(100);
    // 无额外索引配置
});

// Gov_Log 表
builder.Entity<GovLog>(b =>
{
    b.ToTable("Gov_Log");
    b.HasKey(e => e.Id);                         // Guid 主键
    b.Property(e => e.SyncMsg).HasMaxLength(2000);
    b.Property(e => e.SyncResult).HasMaxLength(2000);
});

// Urban_WeighingRecord 表（新增）
builder.Entity<UrbanWeighingRecord>(b =>
{
    b.ToTable("Urban_WeighingRecord");
    b.HasKey(e => e.Id);                         // Guid 主键
    b.Property(e => e.PlateNumber).HasMaxLength(50);
    b.HasIndex(e => e.ClientRecordId).IsUnique(); // 幂等去重索引
});

// AttachmentFile（图片附件）
builder.Entity<AttachmentFile>(b =>
{
    b.ToTable("AttachmentFile");
    b.HasKey(e => e.Id);                         // Guid 主键
    b.Property(e => e.FileName).HasMaxLength(200);
    b.Property(e => e.LocalPath).HasMaxLength(1000);
    b.Property(e => e.AttachType).HasMaxLength(50);
});

// UrbanWeighingRecordAttachment（称重记录-附件关联）
builder.Entity<UrbanWeighingRecordAttachment>(b =>
{
    b.ToTable("Urban_WeighingRecordAttachment");
    b.HasKey(e => e.Id);                         // Guid 主键
    b.HasIndex(e => e.UrbanWeighingRecordId);
    b.HasIndex(e => e.AttachmentFileId);
});
```

## 4. 数据库迁移策略（重建新库）

### 4.1 当前状态

- 使用 SQLite，连接字符串：`Data Source=UrbanManagement.db`
- `AddDefaultRepositories(true)` 自动注册默认仓储
- 支持 EF Core Migrations（尚未生成初始迁移）

### 4.2 迁移策略（放弃旧库，导入新库）

采用一次性重建方案：

| 阶段 | 处理方式 |
|------|---------|
| 创建新库 | 按 Guid 主键目标模型创建全新数据库 |
| 导出旧库 | 从旧 XiaoShan.db 导出 GovProject/GovSyncData/GovLog 等数据 |
| 映射导入 | 按映射规则转换并导入新库（生成新的 Guid 主键） |
| 关联回填 | 按旧键映射关系回填 GovLog → GovSyncData 等外键 |
| 校验切换 | 校验通过后切换应用连接串到新库 |

### 4.3 旧数据导入规则

| 新表 | 旧数据来源 | 关键规则 |
|------|-----------|---------|
| GovProject | Gov_Project | `Id=Guid.NewGuid()`；旧 `ProId` 作为业务字段保留 |
| UrbanWeighingRecord | 旧称重/同步来源 | 以客户端上报语义重建记录，`IsAnomaly` 保留 |
| GovSyncData | UrbanWeighingRecord + 扩展字段 | 同构映射；仅承载业务载荷，不承载同步状态字段 |
| GovLog | Gov_Log | `Id=Guid.NewGuid()`；`SyncId` 通过旧键映射到新 `GovSyncData.Id` |
| AttachmentFile | 旧 SnapImages 路径 | 每个路径生成一条附件，`AttachType=Lrp`，`Id=Guid.NewGuid()` |
| UrbanWeighingRecordAttachment | 旧记录与路径关系 | 按旧记录与图片路径建立关联，主键 Guid |

> 说明：迁移后不再依赖旧库主键类型和主键值，旧库仅作为导入数据源与回溯依据。

### 4.4 配置迁移
